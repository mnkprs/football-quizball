Pre-Launch UX Strategy: 3 Approaches

Based on a comprehensive audit of Stepover's current state. Click to select your preferred approach.

Audit Snapshot

DONE
Streaks, Match History, ELO Tiers, Achievements, Share/Invite, Onboarding, Delete Account, Export Data
GAPS
Empty States (50%), Error Recovery (no retry), Loading (spinners only), Settings (no password/theme/notifs), Stats (partial modes)
MISSING
Haptic/Sound (0%), Push Notifications (0%), Offline Mode (0%), Friends List, Weekly Challenges, Referral System, App Store Assets
A

Polish What Exists

Ship faster. Make current features feel premium.

Scope: Skeleton loaders, haptic/sound feedback, empty states, retry on errors, theme toggle in settings, share game results, app store screenshots
Impact: App feels 2x more polished. First impressions dramatically better.
Doesn't include: Push notifications, friends, referrals, weekly challenges
Risk: Users try it, love the feel, but no hooks to bring them back beyond streaks
B

Retention First

Engagement-focused. Build the hooks that bring users back.

Scope: Push notifications (daily reminders, duel challenges), referral system, weekly challenges/quests, share results (viral loop), friend leaderboards
Impact: DAU/MAU ratio improves. Organic growth via referrals and sharing.
Doesn't include: Skeleton loaders, haptics/sound, empty state polish, settings cleanup
Risk: Retention features on an unpolished base. Users hit rough edges before reaching the hooks.
C

Comprehensive Pre-Launch (Phased)

Do it all, in priority order. Ship when it's right.

Phase 1 - Feel: Haptic/sound, skeleton loaders, empty states, error retry, settings cleanup
Phase 2 - Retain: Push notifications, weekly challenges, referral system
Phase 3 - Grow: Share results with OG previews, friend system, deep link previews
Phase 4 - Launch: App store assets (screenshots, preview video, ASO metadata)
Impact: Complete, premium-feeling app with retention + growth built in from day one
Risk: Larger scope, but no deadline pressure makes this viable

Since you have no hard deadline and want comprehensive coverage, I recommend Option C — phased approach ensures each layer builds on the previous one.
