Phase 3: Grow

Viral loops and social features that drive organic acquisition.

1

Share Results

What: Shareable result cards after every game session
Surfaces:
• Solo finish — "I scored 8/10 and climbed to Gold tier on Stepover!"
• Duel win — "Beat @opponent 7-4 in a football quiz duel"
• Battle Royale — "Survived 8 players, finished #1"
• Logo Quiz — "Identified 15 club crests in a row"
• Achievement unlock — "Just earned Quiz Legend on Stepover"
Format: Generated image card (canvas-based, branded with Stepover logo + app store link). Native share sheet via Capacitor Share plugin. Clipboard fallback for text-only.
OG Preview: Backend endpoint that renders dynamic OG images for shared links (e.g., `/og/result/:id`) so links look rich on social media/messaging apps.
Scope: 1 shared ResultCardComponent, canvas image generator, backend OG endpoint, share button on all result screens
2

Friend System

What: Lightweight friends list with activity and direct challenge
Features:
• Add friend — by username search or share a friend code
• Friend leaderboard — separate tab on leaderboard page (global vs friends toggle)
• Challenge friend — tap a friend to start a duel directly (skips matchmaking queue)
• Activity feed — NOT a full social feed. Just: "Alex reached Diamond tier", "Sam is on a 14-day streak" on the friends tab
DB: `friendships` table (user_id, friend_id, status: pending/accepted, created_at). Symmetric — both users see each other.
Privacy: Friend requests require acceptance. No public friend counts. Block option.
Scope: Backend CRUD module, friend list UI, leaderboard tab, challenge shortcut
3

Deep Link Previews

What: Rich previews when Stepover links are shared in iMessage, WhatsApp, Twitter, etc.
Links that need previews:
• Duel invite — "Challenge from @player — tap to play" + Stepover branded card
• Referral link — "Join Stepover — Football quiz with friends" + download CTA
• Result share — Dynamic score card image (from Share Results above)
• Profile link — Player name, tier badge, ELO
How: Server-side rendered HTML with OG meta tags at each sharable route. Firebase Dynamic Links or custom domain redirect (stepovr.app) that routes to app or store.
App Links: Configure iOS Universal Links + Android App Links so tapping a link opens the app directly if installed, or the store if not.
Scope: Backend OG renderer, Universal Links config (apple-app-site-association), Android assetlinks.json, link routing logic
4

Profile Enhancements

What: Make profiles worth visiting and sharing
Additions:
• Per-mode stats — fill gaps for Duel (W/L/D, win rate), Logo Quiz (best streak, accuracy), News (rounds played, avg score), BR (wins, avg placement)
• Share profile — button to share your profile link (with OG preview from above)
• Weekly challenge badges — display completed challenges from Phase 2
• Favorite mode badge — auto-calculated from most-played mode
Scope: Profile component updates, backend stats aggregation queries, ~4 new stat cards
