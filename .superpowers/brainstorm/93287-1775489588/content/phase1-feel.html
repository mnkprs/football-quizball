Phase 1: Feel

Make the current app feel premium. 6 workstreams, each independently shippable.

1

Haptic & Sound Feedback

What: Add tactile + audio feedback to quiz interactions
Where: Answer tap (light haptic), correct answer (success chime + medium haptic), wrong answer (error buzz + soft thud), timer running out (warning vibrate), achievement unlock (celebration haptic + fanfare), streak milestone (special sound)
How: Capacitor Haptics plugin (already available) + small audio sprite (single file, ~50KB). Sound toggle in settings (default ON). Haptic follows system setting.
Scope: ~15 haptic trigger points, ~8 distinct sounds, 1 settings toggle
2

Skeleton Loaders

What: Replace mat-spinners with content-shaped skeleton placeholders
Where: Profile page (avatar + stats + history), Leaderboard (list rows), Home page (mode cards), Duel/BR lobby (room list), Question loading
How: Single reusable SkeletonComponent with CSS shimmer animation. Variants: line, circle, card, row. No external deps.
Scope: 1 shared component, applied to ~8 pages
3

Empty States

What: Replace hidden sections with friendly illustrations + CTAs
Where: Achievements (none earned yet), Match history (no games), Duel lobby (no active duels), Leaderboard (mode with no players), Profile stats (mode not played yet)
How: Shared EmptyStateComponent with icon, title, subtitle, optional CTA button. Each usage gets contextual copy ("No duels yet — challenge someone!").
Scope: 1 shared component, ~10 integration points
4

Error Recovery

What: Add retry buttons + contextual error messages across the app
Where: Any API-dependent screen (profile, leaderboard, lobbies, question fetch). Network loss during a game (reconnect prompt instead of silent failure).
How: Shared ErrorStateComponent (icon + message + retry button). Wrap critical API calls with retry logic (exponential backoff, max 3). Toast for transient errors, inline for page-level failures.
Scope: 1 shared component, update error interceptor, ~8 integration points
5

Settings Cleanup

What: Complete the settings menu with missing controls
Add: Theme toggle (dark/light — service exists, no UI), Sound on/off toggle, Haptic on/off toggle, Change password (Supabase magic link flow), Notification preferences (placeholder until Phase 2)
Keep: Delete account, export data, sign out (already working)
Scope: 5 new toggles/actions in existing settings-menu component
6

Hide Deferred Modes

What: Properly hide Blitz from the home page instead of "coming soon" tag
Why: App Store reviewers flag non-functional features. "Coming soon" on a paid app feels unfinished.
How: Feature flag or simply remove the card from home. Keep the route/component for future.
Scope: 1 card removal, ~10 minutes
