Phase 2: Retain

Hooks that bring users back. Build on the polished base from Phase 1.

1

Push Notifications

What: Firebase Cloud Messaging for iOS + Android via Capacitor
Notification types:
• Streak rescue — "Your 7-day streak expires in 2 hours!" (evening if no game played today)
• Daily round — "Today's news round is live. 10 fresh questions." (at round reset time)
• Duel challenge — "Someone challenged you to a duel!" (when matched with real player)
• Achievement unlock — "You just earned Quiz Master!" (immediate)
• Weekly digest — "You climbed 45 ELO this week. Keep it up!" (Sunday evening)
Backend: NestJS cron jobs + FCM admin SDK. Store device tokens in `user_devices` table.
Settings: Per-category toggles in notification preferences (Phase 1 placeholder becomes real).
Scope: Capacitor Push plugin, backend FCM service, 5 notification triggers, settings UI
2

Weekly Challenges

What: 3 rotating challenges that reset every Monday, giving users mid-term goals beyond daily streaks
Examples:
• "Win 5 duels this week" (competitive)
• "Answer 50 questions correctly" (volume)
• "Play all 3 modes: Solo, Duel, Battle Royale" (exploration)
• "Get 80%+ accuracy in a Solo session" (skill)
• "Identify 20 logos correctly" (mode-specific)
Reward: No currency needed — challenge completion badge on profile + achievement progress. The reward IS the progression.
UI: Small challenge strip on home page (below hero, above mode cards). Tap to expand details. Progress bars per challenge.
Backend: `weekly_challenges` table, cron to generate 3 random challenges Monday 00:00 UTC, progress tracking via existing game completion hooks.
Scope: 1 new component, 1 new backend module, ~12 challenge templates
3

Referral System

What: Invite friends, earn Pro trial days. Already spec'd in subscription-refinement design.
Mechanic: Invite 3 friends who each play 1 game → earn 7-day Pro trial. Referee gets 3-day trial too.
Implementation:
• Unique referral code per user (stored in `profiles.referral_code`)
• Deep link: `stepovr.app/r/{code}` → onboarding with referrer tracked
• `referrals` table tracks: referrer_id, referee_id, status (invited/signed_up/qualified), created_at
• Qualification = referee plays 1+ game (prevents fake signups)
• One-time reward per referrer (not stackable to prevent abuse)
UI: Referral card in profile or settings. Share button with personalized link. Progress indicator (2/3 friends invited).
Scope: 1 backend module, 1 frontend component, deep link handling, fraud guards
4

Streak Enhancements

What: Make the existing streak system more visible and rewarding
Additions:
• Streak flame on home page — persistent flame icon with count, visible without navigating to profile
• Streak milestones — special celebration at 7, 30, 100 days (ties into haptic/sound from Phase 1)
• Streak freeze (Pro only) — 1 free pass per week to skip a day without losing streak. Strong Pro conversion hook.
Scope: Home page UI tweak, 3 milestone achievements, streak freeze logic (backend + frontend)
